# 简历优化知识

## STAR 法则
考察点：是否能把项目经历写清楚。

回答要点：
- Situation：项目背景。
- Task：你负责的任务。
- Action：你采取的技术方案。
- Result：结果和量化收益。

简历改写模板：
- 基于 FastAPI 设计并实现简历分析接口，使用 Pydantic 完成输入校验和响应建模，提升接口稳定性和可维护性。
- 设计 LLM 输出解析和 fallback 机制，对模型非标准 JSON 输出进行兜底处理，避免报告生成流程异常中断。

项目追问：
- 你的项目中最能体现工程能力的模块是什么？
- 有没有量化指标，例如接口耗时、解析成功率、测试覆盖数量？

## 技术项目 Bullet 写法
考察点：是否能突出技术选型、职责和结果。

回答要点：
- 不要只写“使用了某技术”，要写“用它解决了什么问题”。
- 尽量体现架构、稳定性、数据建模、测试、部署。
- 对 AI 项目要强调结构化输出、可追溯、降级机制。

示例：
- 基于 FastAPI + CrewAI 构建多 Agent 简历分析系统，将 JD 分析、简历解析、匹配评分和报告生成拆分为独立任务，提升 Prompt 可维护性。
- 接入 Tavily 搜索构建岗位市场画像，对岗位样本进行过期识别和 freshness_score 降权，提升市场匹配分析可信度。

## 实习面试表达
考察点：是否能把项目讲成真实工程，而不是 Demo。

回答结构：
- 背景：计算机学生投实习时难判断简历和岗位是否匹配。
- 方案：用多 Agent 分析简历/JD，并结合岗位市场画像给出优化建议。
- 难点：LLM 输出不稳定、岗位搜索结果可能过期、长耗时任务体验差。
- 解决：Pydantic 校验、fallback 解析、岗位 freshness_score、异步任务状态表。

常见追问：
- 如果模型输出错误怎么办？
- 如何证明岗位画像不是过期信息？
- 如果用户很多，SQLite 是否还能支撑？
