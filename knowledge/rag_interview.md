# RAG 与知识库知识

## RAG 的作用
考察点：是否理解为什么需要检索增强。

回答要点：
- RAG 是 Retrieval-Augmented Generation，即先检索相关资料，再让模型基于资料生成。
- 它可以减少 Prompt 冗余，让模型看到更相关的上下文。
- 对简历助手来说，RAG 可用于召回面试知识、技能补强建议、项目优化模板。

- 为什么不能把所有知识库文件都塞进 Prompt？
- RAG 能完全解决幻觉吗？为什么不能？

## 轻量 RAG
考察点：是否理解从简单方案开始迭代。

回答要点：
- 第一版可以用 markdown 分块 + 关键词匹配实现。
- 根据目标岗位、JD、简历缺口召回 top_k 片段。
- 不需要 embedding 和向量数据库，部署和调试成本低。

- 为什么当前项目适合先做轻量 RAG？

## 向量数据库
考察点：是否理解向量检索的适用场景。

回答要点：
- 向量数据库把文本转成 embedding，再按语义相似度检索。
- Chroma、FAISS、Milvus 都可以做向量检索。
- 当知识库规模变大、关键词匹配不准确时，可以升级向量检索。

项目追问：

## RAG 结果可追溯
考察点：是否理解可信 AI 输出。

回答要点：
- 检索结果应保留 source、chunk、score。
- 报告可以展示参考知识片段来源，帮助用户判断建议依据。
- 模型输出不能直接照抄知识库，应结合简历和岗位要求生成。
