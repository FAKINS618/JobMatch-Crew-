# Python 后端实习高频知识

## 数据结构与复杂度
考察点：候选人是否理解 list、tuple、dict、set 的使用场景和复杂度。

回答要点：
- list 适合有序可变序列，按下标访问快，频繁中间插入删除成本较高。
- tuple 是不可变序列，适合作为不可变配置、函数多返回值或 dict key。
- dict 基于哈希表，适合键值查询，平均查询复杂度接近 O(1)。
- set 适合去重、集合交并差运算，也基于哈希表。

- 简历解析后如何对技能关键词去重？
- 岗位技能匹配时为什么适合用 set 做交集？

## 装饰器
考察点：是否理解函数是一等对象、闭包和横切逻辑复用。

回答要点：
- 装饰器本质是接收函数并返回新函数的高阶函数。
- 常用于日志、权限校验、性能统计、缓存、重试。
- FastAPI 的路由装饰器会把函数注册成接口处理函数。

## GIL 与并发
考察点：是否理解 Python 多线程、多进程、异步 IO 的差异。

回答要点：
- GIL 让同一进程内多个 Python 线程不能同时执行 Python 字节码。
- IO 密集型任务适合异步或多线程，例如网络请求、数据库访问。
- CPU 密集型任务更适合多进程或外部计算服务。

## async / await
考察点：是否理解协程和事件循环。

回答要点：
- `async def` 定义协程函数，`await` 用于等待异步操作完成。
- 异步适合高并发 IO，不等于自动加速 CPU 计算。
- FastAPI 支持同步函数和异步函数，异步函数适合数据库、HTTP 请求等 IO。
